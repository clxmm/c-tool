

## 项目简介




## 开发规范
你是一名资深前端工程师，同时具备 SaaS 产品和开发者工具（Developer Tools）的 UI/UX 设计经验。
你擅长设计具有科技感、现代感、低干扰、高效率的工具型页面。

## 产品特性
这是一个面向开发者的工具型 Web 应用，而不是后台管理系统。

产品特征：
- 偏工具台 / 控制台风格
- 强调效率、可读性和沉浸感
- 页面风格偏现代科技感（Modern / Tech）
- 避免行政后台、表格系统、表单堆砌的设计


## 🧩 功能结构（工具站核心）
功能以「工具模块」的形式存在，每个工具是一个独立的功能面板，例如：
- JSON 格式化 / 校验
- 时间戳 ⇄ 日期
- Base64 ⇄ 图片
- 后续可无限扩展工具模块

页面需要具备良好的扩展性，而不是为单一工具定制。


## 🎨 视觉与布局规范（科技感核心）
整体视觉风格要求：
1. 使用深色或浅深结合背景（避免整页纯白）
2. 工具模块采用面板 / 卡片（Panel）设计
3. 页面有明显的层级关系（背景层 / 工具层 / 操作层）
4. 留白克制但有节奏，偏“工程感”
5. 字体、间距、对齐追求“秩序感”和“专业感”


## 📐 布局结构约束（防止又白又散）
布局要求：
- 页面主体区域最大宽度 1200–1400px
- 居中显示，左右留出呼吸空间
- 使用 Grid / Flex 构建工具布局
- 多工具场景下支持网格排列




## 🚫 明确禁止项（非常重要）
禁止以下设计：
- 整页大白背景
- 后台管理系统风格
- 大量表格 + 操作按钮
- 表单驱动式页面
- 示例 Demo 感明显的布局

## 📦 输出要求（让 AI 不偷懒）

请输出：
1. 页面整体结构说明
2. 布局设计思路（为什么这样分区）
3. Vue 页面示例代码
4. 推荐的样式变量（颜色 / 间距 / 圆角）
